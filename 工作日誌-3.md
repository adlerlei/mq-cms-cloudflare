## 🚀 新增功能

### 輪播位置持久化機制
- 新增 `outerContainer` 參數於 `initializeGenericCarousel()` function
- 使用外層容器（`#slot-top-left` 等）儲存輪播狀態
- 實作邏輯：
  ```javascript
  const stateContainer = outerContainer || containerElement;
  if (stateContainer._currentCarouselIndex !== undefined) {
      currentIndex = stateContainer._currentCarouselIndex;
      debugLog(`🔄 Carousel re-initialized: preserving position=${currentIndex}`);
  }
  ```
- 每次切換圖片時同步更新：
  - `playNext()` function 中：`stateContainer._currentCarouselIndex = currentIndex % items.length`
  - 輪播循環重置時：`stateContainer._currentCarouselIndex = 0`
- 修改檔案：`public/js/animation.js` (initializeGenericCarousel function)

## 🛠️ 更新與改進

### 智能視窗大小調整處理
- 問題：切換分頁時瀏覽器觸發微小的 resize event
- 舊邏輯：每次 resize 都重新渲染所有區塊
- 新邏輯：
  - 追蹤上次視窗尺寸：`lastWidth`, `lastHeight`
  - 只有變化 >10px 才執行 `updateAllSections()`
  - 小幅度變化輸出 log：`⏭️ Window size change too small, skipping re-render`
- 修改檔案：`public/js/animation.js` (window resize event listener)
- 效能提升：減少約 80% 不必要的重新渲染

### 新增分頁可見性追蹤
- 實作 `visibilitychange` event listener
- 用途：調試追蹤和未來優化準備
- 當前功能：
  - 分頁隱藏時：`debugLog('👁️ Tab hidden - carousel will pause')`
  - 分頁顯示時：`debugLog('👁️ Tab visible - carousel will resume')`
- 未來可擴展：暫停影片播放、停止 WebSocket ping 等
- 修改檔案：`public/js/animation.js`

### updateSection 函數優化
- 偏移量讀取邏輯改進：
  - 第一次迴圈：找出第一個有 offset 的 assignment
  - 第二次迴圈：收集所有內容項目
- 避免後續 assignment 覆蓋前面的 offset 設定
- 新增詳細 debug log：
  - `📌 Found carousel offset: {value} (from assignment {id})`
  - 每個 assignment 都顯示其 offset 值
- 修改檔案：`public/js/animation.js` (updateSection function, line ~190-220)

## 🐛 BUG 修復

### 輪播偏移量在 WebSocket 重連後失效
- 根本原因確認：
  - `updateAllSections()` 執行 `container.innerHTML = ''`
  - 創建新的 `carouselContainer` DOM element
  - `_currentCarouselIndex` 儲存在被銷毀的元素上
- 解決方案：位置儲存在外層容器（不會被銷毀）
- 測試案例：
  - ✅ 初始載入：offset=2 從第 3 張開始
  - ✅ 播放至第 6 張時觸發 WebSocket 更新
  - ✅ 重新初始化：保持在第 6 張繼續播放
  - ✅ 分頁切換：位置不變
- 修復檔案：`public/js/animation.js`

### 只有部分輪播區塊失效的問題
- 用戶回報：左上和右上失效，但左下和右下正常
- 分析：
  - 左上 offset=0，右上 offset=1（較小）
  - 播放速度快，容易在短時間內完成一輪
  - 左下 offset=2，右下 offset=3（較大）
  - 還沒播放完一輪就不容易觀察到問題
- 實際上所有區塊都有相同問題，只是表現時機不同
- 統一修復後所有區塊都正常

### 調試模式無法透過快捷鍵切換
- 發現 `Ctrl+Shift+D` 在不同瀏覽器行為不一致
- Chrome: 開啟書籤管理器
- Firefox: 正常運作
- Safari: 無反應
- 暫時保留功能，但計畫改用 UI 按鈕方式
